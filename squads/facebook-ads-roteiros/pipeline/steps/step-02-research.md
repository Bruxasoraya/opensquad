# Step 02: Weekly Research

## Context Loading

Before executing, load the following context:

- **Input:** `squads/facebook-ads-roteiros/output/research-focus.md` — contains the user-defined research focus, time range, and specific angles from step-01 checkpoint
- **Company context:** Bruxa Soraya Tarot — spiritual services provider (tarot readings, love spells, benzimento, app). Brazilian Portuguese-speaking audience. Facebook Ads as primary paid channel.
- **Agent:** Rafa Referência, Pesquisadora de Tendências (`squads/facebook-ads-roteiros/agents/researcher.agent.md`)
- **Production context:** 15 scripts/week across 5 categories — Tarot Perguntas, Trabalhos Amorosos, Benzimento Segunda, App, Benzimento Diário (3 scripts each)

## Instructions

### Process

1. **Read research-focus.md in full.** Extract the research focus topic, the selected time range, and any specific angle or theme the user indicated. These parameters constrain the search scope — do not research beyond them without flagging the deviation.

2. **Execute `find-trends` task** (`agents/researcher/tasks/find-trends.md`). Run all 4 sub-steps: search for trending hooks in the spiritual Facebook Ads niche, identify Meta/Facebook platform and policy updates, analyze competitor activity in the Brazilian spiritual services market, and map seasonal and astrological angles for the current production week. Apply the user's time range filter to all source citations.

3. **Execute `compile-weekly-brief` task** (`agents/researcher/tasks/compile-weekly-brief.md`). Synthesize all findings from the find-trends task into a structured brief organized by the 5 categories. Ensure all 5 categories have angle recommendations, all compliance warnings are surfaced at the top, and the hook library is populated.

4. **Write output to `weekly-trends-brief.md`.** The compiled brief is the final output of this step. It must be written in valid YAML inside a markdown code block, preceded by a short prose summary (3-5 sentences) of the key findings for quick human review at the next checkpoint.

## Output Format

The output file must follow this structure:

```
# Weekly Trends Brief
**Week:** <date range>
**Generated by:** Rafa Referência
**Research focus:** <extracted from research-focus.md>

## Summary
<3-5 sentence prose summary of key findings — most important hook patterns, any compliance warnings, top seasonal angle, strongest competitor signal>

## Full Brief

\`\`\`yaml
weekly_brief:
  week: "<date range>"
  generated: "<date>"
  compliance_warnings: [...]
  categories:
    tarot_perguntas: {...}
    trabalhos_amorosos: {...}
    benzimento_segunda: {...}

    benzimento_diario: {...}

```

## Veto Conditions

- **Veto if fewer than 4 of the 5 categories have populated angle recommendations.** The step must not write a partial brief. If research yields insufficient data for a category, the agent must attempt a broadened search before declaring a gap.
- **Veto if compliance_warnings section is absent.** This is a structural requirement regardless of whether new warnings were found. If no new warnings exist, the section must confirm "no new policy updates identified this week."

## Quality Criteria

- [ ] Research focus parameters from step-01 are visibly applied (time range, focus topic, specific angle)
- [ ] All 5 categories present in the compiled brief with at least one primary and one secondary angle
- [ ] Prose summary written (3-5 sentences) before the YAML block for quick human review
- [ ] Every hook in the hook library has a source, confidence level, and category tags
- [ ] Compliance warnings surfaced at the top of the brief structure
